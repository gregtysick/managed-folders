# ManagedFolders 2.0

Status: current  
Released: 2026-06-10

ManagedFolders `2.0` is an operating-model release rather than a documentation-only version change.

## Release Focus

- Add post-install health reviews as a second-pair-of-eyes check.
- Keep the Executive layer focused on current state, open loops, next actions, and active projects.
- Separate candidate-work detection from approved task ownership.
- Allow optional structured project-management databases and dashboards without making them required for every folder.
- Keep records and timekeeping evidence separate from current project-management state.
- Treat database-backed records and dashboards as optional implementations with verification requirements when enabled.
- Use a lightweight registry for version, review, and upgrade readiness.
- Add repository-readiness checks for documentation, deployment boundaries, and branch roles.
- Use Markdown configuration files with YAML frontmatter so configuration remains structured and opens normally in knowledge-management tools.

## Release Principle

Version `2.0` represents a reusable operating-model improvement checked against more than one managed workspace.

Private workspace paths, client details, and machine-specific configuration remain outside this public mirror.
