# ManagedFolders 2.1

Status: draft for next public update  
Planned release: 2026-06-15

ManagedFolders `2.1` is a structure and clarity release.

## Highlights

- Clarifies the current Executive structure: `current_state`, `task_radar`, and `project_manager`.
- Adds a fake Example Managed Folder so the system shape can be inspected without exposing private data.
- Expands the public-safe SignalDesk outline so the control-plane concept is easier to understand.
- Keeps the public repo positioned as a documentation and portfolio mirror, not a turnkey installer.

## Changes

- Split the Executive layer into three visible operating areas:
  - `current_state` for active operating truth.
  - `task_radar` for candidate work that still needs review.
  - `project_manager` for approved work and readable task views.
- Keep user documents, meeting packets, deliverables, durable facts, and project files outside the Executive layer.
- Add `docs/managedfolders-2.1.md` as the current public operating-model explanation.
- Add `examples/Example Managed Folder/` as a fake, public-safe structure demo.
- Add lightweight `signaldesk/` subfolder indexes for the public control-plane outline.

## Public Safety

- No private paths.
- No private names.
- No client data.
- No machine-specific configuration.
- Example data is invented.

## Upgrade / Alignment Notes

- `2.1` should be published after the `2.0` history is already visible.
- The repo-local release note lives here in `docs/2.1.md`.
- A GitHub Release can later reuse this note when a formal public release artifact is wanted.

## Release Principle

Version `2.1` makes the current system shape easier to inspect.

This repository is still a public mirror and portfolio artifact, not a turnkey installer.
