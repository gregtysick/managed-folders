# ManagedFolders 2.0

ManagedFolders 2.0 is the shift from passive folders to active operating folders.

It describes a public-safe model for organizing work so folders can help track what matters now, what was learned, what remains true, what may be getting missed, and what work has been approved.

## Purpose

The purpose of ManagedFolders 2.0 is to make folders more useful as operating systems for real work.

The model separates current execution, historical evidence, durable knowledge, candidate work, and approved work into clearer layers.

## Core Layers

### Executive

The Executive layer holds current state, decisions, priorities, open loops, next actions, and project or task management supervision.

It answers the question: what matters now?

### Scribe

The Scribe layer holds chronological history, journals, communications, logs, imports, source evidence, and the record of how information was learned.

It answers the question: how did we know this?

### Admin / Knowledge

The Admin / Knowledge layer holds durable facts, stable reference knowledge, people and context, systems, operations, and brand or communication standards.

It answers the question: what remains true?

### Task Radar

Task Radar is a proactive open-loop detection layer.

It is designed to scan communications, journals, project notes, documents, and daily activity to surface possible tasks, follow-ups, blockers, unresolved questions, and missing decisions before they are lost.

### Project Manager

The Project Manager layer is the structured task and project system.

It stores approved work, priorities, dependencies, statuses, and work-session context, while Markdown remains the readable interface.

## Operating Principle

Executive = what matters now.

Scribe = how we learned it.

Admin = what remains true.

Task Radar = what might be getting missed.

Project Manager = what is officially approved work.

## Current Status

ManagedFolders 2.0 is under active development and is being documented through public-safe updates.
